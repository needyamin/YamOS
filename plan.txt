YamOS major implementation plan
===============================

Investigation pass
------------------

Date: 2026-05-09
Scope: process ABI, execve, fd lifetime, task cleanup, mmap, signals, threads,
and the next steps that make YamOS feel powerful in current years.

Core rule
---------

Do not fake capability in userland. If a runtime, package manager, browser, or
tool needs a kernel feature, implement the kernel/ABI feature first. Returning
honest failure is better than returning fake success.

What the investigation found
----------------------------

1. execve is no longer just a plan.
   SYS_EXECVE now routes into elf_exec_resolved_argv_envp(). Static ELF exec is
   present in source, and PT_INTERP exec has a first implementation that maps the
   main ELF plus interpreter, builds argv/envp/auxv, seeds brk, switches pml4,
   and jumps into user mode.

2. The exec path is now boot-probed for the core static contract.
   The init-time /bin/exec-test probe proves failed exec returns and the old
   process continues, successful exec keeps the same PID, cwd survives, normal
   fds survive, and FD_CLOEXEC fds close. Remaining work is stress coverage,
   old-mapping counter proof, and dynamic-loader/TLS behavior.

3. Dynamic linking is still shallow.
   The loader path has auxv and PT_INTERP structure, but relocation support is
   narrow, TLS program headers are not complete, and there is no proven small
   dynamically linked libc test yet. This is the next big compatibility wall.

4. Process memory cleanup improved.
   vmm_destroy_user_pml4() walks user page tables and releases pages through
   page refcounts. execve destroys the replaced address space. brk is now tracked
   per task and shrink unmaps released heap pages.

5. Process lifetime cleanup is halfway done.
   waitpid reaping now releases child VMA metadata, non-shared user pml4, FPU
   state, fd table resources, kernel stack pages, YamGraph task nodes, and the
   task object. Forked tasks now receive cloned VMA metadata and their own
   YamGraph node instead of inheriting the parent's node by memcpy. Basic
   build/boot proof now passes; missing pieces remain: targeted cleanup-counter
   boot tests and correct full process-vs-thread ownership. Scheduler lifetime
   and cleanup counters are now exposed through SYS_SCHED_INFO and the terminal
   `sched` command. Orphan handling now detaches children from exiting parents
   and queues parentless dead tasks for deferred cleanup once they are no longer
   active.

6. FD lifetime is now real enough for exec work.
   fork/thread creation retains copied fd_table entries. waitpid closes child
   fds. task_t has per-fd descriptor flags. SYS_FCNTL supports F_GETFD/F_SETFD.
   dup/dup2 clear descriptor flags on the new fd. execve closes FD_CLOEXEC fds.

7. Threads exist but are not a full process model.
   SYS_THREAD_CREATE creates tasks sharing the parent pml4, fd table entries are
   retained, and pthread uses it. But thread groups are not fully cleaned up on
   exec/exit, joins are waitpid-shaped, user stacks/TLS ownership are weak, and
   killing or execing a multithreaded process is intentionally restricted.

8. Signals are a minimal bridge, not POSIX-correct yet.
   sigaction/sigprocmask exist, but delivery is a lightweight handler jump with
   a pushed resume address. There is no real sigframe/ucontext/rt_sigreturn,
   no robust interrupted-syscall restart model, and no complete process/thread
   targeting semantics.

9. mmap is not ready for serious ports.
   Anonymous lazy mapping exists through VMAs and page faults, but file-backed
   mmap does not actually map file contents with MAP_PRIVATE/MAP_SHARED
   semantics. The mmap base is global instead of per-process, which can become
   a correctness problem as exec/fork usage grows.

10. Networking is useful but not browser/runtime grade.
    TCP sockets, nonblocking behavior, select, DNS, DHCP, and plain HTTP are in
    tree. sendto/recvfrom still return failure in the syscall dispatcher, Unix
    sockets are missing, socket options are limited, and TLS/HTTPS is not a full
    verified handshake yet.

11. The README is behind the newest process work.
    README still describes full execve as missing. That was true before the
    recent slice, but the current tree now has first execve/PT_INTERP work that
    must be described as "implemented but unverified" after build testing.

Pre-plan: highest intelligence path
-----------------------------------

The best route is not "add more features." The best route is to make the
process ABI trustworthy. Once process replacement, lifetime cleanup, mmap,
signals, and dynamic loading are reliable, every later subsystem becomes easier:
shells, package manager, language runtimes, browser work, and test harnesses.

Gate 1: Process lifetime closure
--------------------------------

Goal:
Make task creation, exec, exit, and waitpid stop leaking or destroying the wrong
resources.

Tasks:
- Add vmm_free_kernel_stack() matching vmm_alloc_kernel_stack(). Status:
  implemented in source.
- Free child kernel stacks during waitpid reap. Status: implemented in source.
- Destroy YamGraph task nodes during task cleanup. Status: implemented in source.
- Create one cleanup helper for final task destruction so waitpid, failed fork,
  failed thread create, and future kill paths do not duplicate partial cleanup.
  Status: first helper implemented for spawn/fork/thread failure and waitpid
  reap paths.
- Clone forked VMA metadata so waitpid does not free the parent's VMA list.
  Status: implemented in source.
- Add cleanup counters or boot logs for repeated fork/exec/wait loops. Status:
  source-level counters implemented for task creation, fork, thread creation,
  reaping, task-object frees, kernel-stack frees, user-pml4 destruction,
  VMA-list destruction, fd-table closure, and YamGraph node destruction. The
  counters are printed by sched_print_stats(), copied through SYS_SCHED_INFO,
  and visible in the compositor terminal with `sched`.
- Define and implement orphan/dead-task cleanup for tasks that cannot be reaped
  through the current parent waitpid path. Status: first source-level path
  implemented. Exiting parents detach children; already-dead children and
  parentless dead tasks are queued for deferred reaping; forced-dead runqueue
  tasks are unlinked and cleaned instead of just dropping the task count. Shared
  thread address spaces are retained conservatively until full thread-group
  teardown exists.
- Split "process-owned" resources from "thread-owned" resources:
  - process-owned: pml4, fd table references, cwd, brk, process signal state
  - thread-owned: kernel stack, user stack ownership, FPU state, tls_base,
    pending thread signal state
- Define what happens when a process leader exits while threads still exist.
- Define what happens when a non-leader thread exits or is joined.

Exit criteria:
- Reaping a process releases pml4, VMAs, fds, FPU state, kernel stack, YamGraph
  node, and task object exactly once. Source path implemented; basic build/boot
  passes; init asserts scheduler deltas from `/bin/exec-test` (fork/exec burst plus
  exec ABI checks) and orphan counters after `/bin/orphan-test`. Kernel-page-table
  identity for `sched`/`vmm` must use the boot kernel PML4 snapshot, not CR3 (CR3 may
  still be user while switching to kernel threads with NULL `task_t.pml4`).
- Reaping a thread does not destroy the shared process address space. Source
  path now avoids destroying shared pml4/VMA metadata; thread-group semantics
  still need stronger policy.
- Repeated fork/exec/wait loops do not grow task, fd, or page counts.

Gate 2: execve correctness contract
-----------------------------------

Goal:
Turn execve from "first implementation" into a kernel contract that tests can
trust.

Tasks:
- Verify static execve: /bin/exec-test calls execve("/bin/hello"). Status:
  init-time boot probe passes.
- Confirm exec success preserves PID and parent relation but replaces the image.
  Status: same-PID replacement passes in the boot probe.
- Confirm exec failure returns -1 and the caller continues in the old image.
  Status: missing-path exec failure passes in the boot probe.
- Confirm cwd survives exec. Status: boot probe passes with /tmp.
- Confirm non-CLOEXEC fds survive exec. Status: boot probe passes.
- Confirm FD_CLOEXEC fds close on exec. Status: boot probe passes.
- Confirm signal handlers and pending signals reset according to YamOS rules.
- Confirm brk starts after the new image, not the old one.
- Confirm old pml4 and old VMAs are destroyed after the new address space is
  live.
- Keep exec blocked from non-leader threads until thread-group cleanup exists.

Exit criteria:
- A boot/user test proves static exec, failed exec, cwd, fd inheritance,
  FD_CLOEXEC, argv/envp, and same-PID behavior.

Gate 3: Dynamic ELF loader smoke test
-------------------------------------

Goal:
Run one small dynamically linked YamOS program before pretending to support
large ports.

Tasks:
- Build or import a tiny ld.so-compatible test target.
- Complete PT_TLS program-header handling.
- Add the missing auxv entries expected by the chosen libc/loader.
- Expand relocations beyond R_X86_64_RELATIVE as required by the test loader.
- Verify AT_PHDR, AT_PHENT, AT_PHNUM, AT_ENTRY, AT_BASE, AT_RANDOM, AT_PAGESZ,
  and AT_EXECFN values in userspace.
- Add a dynamic exec test separate from static /bin/exec-test.

Exit criteria:
- A tiny dynamically linked program runs from execve.
- The same program can call write, brk/malloc, and exit.

Gate 4: mmap and file-backed memory
-----------------------------------

Goal:
Make mmap useful for loaders, libc, package tools, and future browser/runtime
ports.

Tasks:
- Move next_mmap_addr into task_t so mappings are per-process.
- Implement MAP_FIXED validation and collision handling.
- Track file-backed VMAs with file references and offsets.
- Implement MAP_PRIVATE lazy file page-in.
- Implement MAP_SHARED write-back policy or return honest failure until it is
  ready.
- Make munmap handle partial unmaps or reject them consistently.
- Ensure fork preserves VMA metadata and CoW semantics.

Exit criteria:
- mmap anonymous, mmap file private, mprotect, munmap, fork, and exec have tests.

Gate 5: Signals and threads
---------------------------

Goal:
Support real pthread and Unix program behavior instead of only demos.

Tasks:
- Implement real sigframe/ucontext layout.
- Implement sigreturn restore instead of the current lightweight return-address
  trick.
- Define process-directed vs thread-directed signal delivery.
- Implement SIGCHLD behavior for process children.
- Add syscall interruption/restart rules.
- Harden pthread_join and thread exit around thread groups.
- Track user stack/TLS allocations so thread cleanup can release them or expose
  a clear ownership rule.

Exit criteria:
- Tests cover signal handler return, blocked signals, kill/raise, child exit
  SIGCHLD, pthread_create/join, mutex/condvar, TLS keys, and thread exit.

Gate 6: Networking/TLS/package foundation
-----------------------------------------

Goal:
Only build package/browser dreams on real transport and security.

Tasks:
- Implement sendto/recvfrom or explicitly split UDP from TCP in the ABI.
- Add getsockname/getpeername, shutdown, and useful socket options.
- Add Unix-domain sockets for local service IPC compatibility.
- Finish mbedTLS-backed HTTPS with certificate chain validation, hostname
  verification, time validation, and root store handling.
- Add signed ypkg repository metadata and package signatures.

Exit criteria:
- A package metadata fetch uses verified HTTPS and rejects bad certificates or
  bad signatures.

Do not do next
--------------

- Do not add fake app installers or fake package success.
- Do not claim browser-class support before dynamic linking, signals, mmap,
  TLS, and process lifetime are reliable.
- Do not widen the syscall ABI randomly; strengthen the semantics already
  exposed.
- Do not enable full SMP task scheduling before process/address-space cleanup
  and TLB shootdown behavior are audited.
- Do not rewrite unrelated desktop/application code while the process ABI is
  still settling.

Bottom line
-----------

YamOS becomes powerful by becoming boringly correct at the Unix-like foundations:
exec, wait, fd lifetime, mmap, dynamic loading, signals, threads, TLS, and tests.
The current code is moving in the right direction. The next intelligent move is
to verify, close leaks, and turn the process ABI into a contract.
