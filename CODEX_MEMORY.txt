Current capability (summary - detail drifts; verify in tree)
------------------------------------------------------------
- QEMU path: PCI inventory, FAT32 block-backed mounts under /mnt, VFS promotion
  of persistent dirs when vd0 present, e1000 + DHCP/DNS/TCP/HTTP.
- TLS: store bootstrap + ClientHello probe; full HTTPS handshake not done.
- execve foundation in progress: SYS_EXECVE routes to elf_exec_resolved_argv_envp,
  static ELF exec works in source and now has an init-time /bin/exec-test probe
  for failed exec, same-PID replacement, cwd preservation, fd inheritance, and
  FD_CLOEXEC. PT_INTERP exec maps main+interpreter, builds auxv stack, and
  enters via enter_user_mode_ldso. Still needs more relocations, TLS
  program-header support, and real dynamic-linker tests.
- Process/memory cleanup improved: vmm_destroy_user_pml4 walks full user page
  tables and uses pmm_page_unref for mapped pages; execve frees replaced pml4.
  task_t has per-process brk_start/brk_current; spawn/exec seed brk from ELF end;
  sys_brk is no longer global and shrink unmaps heap pages.
- FD lifetime improved: fork/thread creation retain copied fd_table entries;
  waitpid reaping closes child fd table and releases VMA metadata, non-shared
  pml4, FPU state, and task object. task_t has per-fd descriptor flags;
  SYS_FCNTL supports F_GETFD/F_SETFD; dup/dup2 clear descriptor flags; execve
  closes FD_CLOEXEC descriptors; /bin/exec-test now exercises that flag before
  exec. Process lifetime cleanup now has vmm_free_kernel_stack(), a centralized
  scheduler task-resource cleanup helper, forked-task VMA metadata cloning, fork
  YamGraph node creation, and waitpid reaping that frees kernel stacks and
  destroys task YamGraph nodes. Unix fork semantics are exercised end-to-end
  (parent gets child PID, child syscall returns 0) via syscall_resume_rsp,
  fork_child_trampoline, and a bootstrap frame placed below the duplicated
  syscall_entry snapshot so saved user RIP is not clobbered. This slice builds
  and boots to userspace.
  Scheduler lifetime/cleanup counters are exposed through SYS_SCHED_INFO and the
  compositor terminal `sched` command. Orphan handling now detaches children from
  exiting parents and queues parentless dead tasks for deferred cleanup after
  they are no longer active. Full thread-group ownership/teardown still pending;
  shared thread address spaces are retained conservatively.
- Threads/signals/futex/pthread pieces and SYS_SELECT/SYS_FCNTL nonblocking I/O:
  see syscall.h + Phase notes in README.
- Desktop: compositor-native Wayland-style UI, multi-user, File Manager, browser
  plain-HTTP path, and settings windows. See README "Current Desktop Behavior".
- Major gaps: complete dynamic linker/TLS ABI, full signals, full TLS/HTTPS,
  portable Linux userspace, browser-class stack.

Roadmap (compressed)
--------------------
Phases 1-2 complete; Phase 3 in progress. Current local plan in plan.txt starts
with process ABI: execve, PT_INTERP, memory/fd lifetime, then TLS program
headers, sigframe/ucontext, fuller dynamic linker, full TLS/HTTPS, ypkg,
fork/Unix sockets, virtio-gpu/Mesa path, libc port, eventual Chromium-class browser.
